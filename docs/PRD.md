## 1. 产品概述

### 1.1 产品定位

SocialMedia-Hub 是一个**高性能、多平台社交媒体数据 API 服务**，提供统一的 REST API 接口，支持 16+ 社交媒体平台的数据访问，包括抖音、TikTok、Instagram、YouTube、Twitter/X、小红书、B站、微博等。

### 1.2 目标用户

| 用户类型 | 使用场景 |
|----------|----------|
| **开发者** | 构建社交媒体数据分析应用 |
| **数据科学家** | 收集社交媒体数据用于机器学习训练 |
| **AI 工程师** | 为 LLM 提供社交媒体上下文数据 |
| **营销团队** | 监测品牌舆情、竞品分析 |
| **内容创作者** | 分析内容表现、粉丝画像 |
| **研究人员** | 社交媒体趋势研究、学术分析 |

### 1.3 核心价值

1. **统一接口** — 一个 API Key 访问 16+ 平台
2. **1068+ 端点** — 覆盖各平台核心功能
3. **1000 MCP 工具** — Model Context Protocol 集成
4. **智能获取** — yt-dlp + API 自动切换
5. **语音识别** — Whisper 语音转文字
6. **反爬保护** — 随机延迟、Cookie 轮换、代理池
7. **生产级质量** — 类型安全、自动重试、结构化异常

## 5. API 设计

### 5.1 通用响应格式

```json
{
  "code": 200,
  "request_id": "uuid",
  "router": "/api/v1/{platform}/{endpoint}",
  "params": {},
  "data": {},
  "time": "ISO8601",
  "time_zone": "UTC"
}
```

### 5.2 错误响应格式

```json
{
  "detail": {
    "code": 401,
    "request_id": "uuid",
    "message": "API token is missing",
    "message_zh": "API令牌缺失",
    "router": "/api/v1/{platform}/{endpoint}",
    "params": {},
    "headers": {},
    "time": "ISO8601",
    "time_stamp": 1234567890,
    "time_zone": "America/Los_Angeles"
  }
}
```

### 5.3 认证方式

```
Authorization: Bearer <API_KEY>
```

### 5.4 端点命名规则

| 规则 | 示例 |
|------|------|
| 路径格式 | `/api/v1/{platform}/web/{function}` |
| 平台标识 | 小写，如 `douyin`, `tiktok`, `instagram` |
| 接口类型 | `web` (网页端) / `app` (移动端) |
| 功能命名 | 动词_名词，如 `fetch_video`, `fetch_user_profile` |

### 5.5 分页参数

| 参数 | 类型 | 说明 |

| `page` | int | 页码分页 |
